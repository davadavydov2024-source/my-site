# Blade Shop

Полнофункциональный маркетплейс игровых предметов: Next.js 14 + TypeScript + TailwindCSS + Firebase + EmailJS.

Всё, что описано ниже, реально работает (не демо/заглушки) при условии, что Firebase настроен по инструкции.

---

## ⚡ Частая ошибка: "Missing or insufficient permissions"

Если при регистрации/входе видишь `FirebaseError: Missing or insufficient permissions` — это значит, что правила Firestore (`firestore.rules`) ещё не опубликованы в консоли Firebase, и сейчас действуют правила по умолчанию ("запретить всё").

Чек-лист:
1. Зайди в https://console.firebase.google.com/project/blade-shoop/firestore/rules (важно — именно `/firestore/rules`, не `/database/.../rtdb/`, см. раздел 3 ниже про путаницу с Realtime Database)
2. Посмотри, что сейчас написано в редакторе правил
3. Выдели всё (Ctrl+A), удали, вставь содержимое файла `firestore.rules` из этого архива
4. Нажми **Publish**
5. Подожди 1-2 минуты, обнови страницу сайта и попробуй снова

## Что внутри

**Магазин:**
- Главная (hero + популярные игры + рекламные баннеры)
- Каталог с фильтрами (поиск, редкость, наличие, сортировка)
- Страница товара, корзина (localStorage), промокоды (демо-коды: `BLADE10`, `WELCOME5`)
- Форма «Продать предметы»
- **Каталог и список игр изначально пустые** — никаких фейковых товаров. Всё добавляется через админ-панель.

**Аккаунт:**
- Регистрация/вход email+пароль и через Google, подтверждение email, восстановление пароля
- Личный кабинет: баланс, история заказов, безопасность (смена пароля, повторная отправка письма подтверждения)
- Пополнение/вывод баланса через заявку → подтверждение в Telegram-боте администратора

**Админ-панель** (`/admin`, доступна только аккаунтам из списка админов):
- Дашборд: аналитика (пользователи, заказы, доход) + график дохода за 14 дней
- Пользователи: поиск, изменение баланса, бан/разбан, цветные метки
- Товары: полный CRUD (создание/редактирование/удаление)
- Игры: полный CRUD (название, slug, изображение) — это категории, по которым фильтруется каталог
- Заявки на баланс: подтверждение/отклонение пополнений и выводов
- Реклама: полный CRUD баннеров (заголовок, описание, цвет, кнопка, ссылка, приоритет, срок действия, вкл/выкл) — баннеры реально показываются на сайте
- Рассылка: отправка письма всем пользователям через EmailJS

## Сознательно не реализовано в этом каркасе

Чтобы не делать фальшивый, нерабочий функционал ради галочки, следующее **не включено**, а не имитировано:
- 2FA, история устройств и IP — требует серверной части (Cloud Functions) и стороннего гео-IP сервиса
- Отзывы, история изменения цены товара — легко добавить по аналогии с уже готовым CRUD товаров
- Тикет-система поддержки — пока используется прямой переход в Telegram

Если что-то из этого нужно — скажи, добавим по тому же принципу (реальная Firestore-коллекция + рабочий CRUD), а не заглушку.

## Установка

### 1. Зависимости

```powershell
npm install
```

### 2. Файл окружения

В PowerShell `cp` с двумя аргументами иногда не создаёт файл при копировании через буфер обмена — используй именно эту команду:

```powershell
Copy-Item .env.local.example .env.local
```

Проверь, что файл правда создался и не пустой:

```powershell
Get-Content .env.local
```

Все нужные ключи (Firebase, EmailJS) уже вписаны в `.env.local.example`, переименовывать значения не нужно.

### 3. Включить Firestore в Firebase (самый частый источник ошибок)

⚠️ **Важно не перепутать.** В Firebase Console есть два разных продукта с похожими названиями:
- **Firestore Database** — это база данных, которую использует этот проект (`getFirestore()` в коде)
- **Realtime Database** — другой, более старый продукт, проекту не нужен, в коде не используется

Если ты откроешь Rules у Realtime Database и попытаешься вставить туда правила Firestore — будет ошибка `Parse error`, потому что у них разный синтаксис. Открывай именно Firestore.

Шаги:
1. Зайди на https://console.firebase.google.com/project/blade-shoop/firestore
2. Если видишь кнопку **Create database** — нажми её, выбери **Start in production mode**, выбери регион (любой, например `eur3`), нажми **Enable**
3. Если база уже создана — просто убедись, что попал в раздел **Firestore Database**, а не Realtime Database (смотри на адрес страницы: должно быть `/firestore/`, а не `/database/.../rtdb/`)
4. Перейди во вкладку **Rules** внутри Firestore (адрес вида `.../firestore/rules`)
5. Открой файл `firestore.rules` из этого архива, скопируй его содержимое целиком
6. В консоли Firebase выдели всё в редакторе правил (Ctrl+A) и удали, затем вставь скопированное
7. Нажми **Publish**

Если видишь `Parse error: Line 1` — почти наверняка ты в Realtime Database, а не в Firestore. Проверь адрес страницы.

### 4. Сделать себя админом

1. Запусти сайт (`npm run dev`) и зарегистрируйся обычным способом через форму на сайте
2. В Firebase Console → **Authentication** → **Users** найди свою запись и скопируй её **UID**
3. Открой `.env.local`, впиши:
   ```
   NEXT_PUBLIC_ADMIN_UIDS=твой_uid_сюда
   ```
   (через запятую, если админов несколько)
4. Открой `firestore.rules`, найди строку:
   ```
   function isAdmin() {
     return isSignedIn() && request.auth.uid in [
       "REPLACE_WITH_ADMIN_UID_1"
     ];
   }
   ```
   замени `REPLACE_WITH_ADMIN_UID_1` на свой реальный UID, сохрани файл, и снова опубликуй правила в консоли Firebase (шаг 3, пункты 5-7)
5. Перезапусти `npm run dev` — в личном кабинете появится пункт «Админ-панель»

### 5. (Опционально) Добавить категории игр

Каталог по умолчанию полностью пустой — это сделано намеренно, чтобы не было фейковых демо-товаров. Все товары и игры ты добавляешь сам через `/admin` → Игры / Товары.

Если хочешь не вводить вручную типовой список игр (Roblox, Grow a Garden, Adopt Me и т.д.), а сразу получить готовый список категорий — можно один раз запустить:

```powershell
npm run seed
```

Это создаст только **категории игр** (без единого товара). Если в этот момент правила Firestore ещё не дают анонимной записи — временно поставь `allow read, write: if true;` на всё (см. ниже), запусти сидирование, затем сразу верни нормальные правила из `firestore.rules`.

**Временные правила только для разового сидирования** (обязательно верни боевые правила сразу после):
```
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /{document=**} {
      allow read, write: if true;
    }
  }
}
```

### 6. Запуск

```powershell
npm run dev
```

Откроется на http://localhost:3000.

## Деплой в продакшн

Самый простой вариант — [Vercel](https://vercel.com) (создатели Next.js):
1. Залей проект на GitHub
2. Импортируй репозиторий в Vercel
3. В настройках проекта Vercel добавь все переменные из `.env.local` (Settings → Environment Variables)
4. Deploy

После деплоя не забудь добавить домен Vercel в Firebase Console → Authentication → Settings → **Authorized domains**, иначе вход через Google не будет работать на проде.

## ⚠️ Важно про схему оплаты

Пополнение/вывод баланса сейчас работает через ручную заявку: пользователь создаёт заявку → пишет администратору в Telegram-бота → админ присылает реквизиты → подтверждает заявку в админке, после чего баланс меняется автоматически.

Это рабочая, но полностью ручная схема без независимого подтверждения платежа — что создаёт риск:
- споров «я перевёл деньги, баланс не обновили»;
- путаницы из-за бот-двойников (пользователю в коде явно показано предупреждение: переводить деньги только по реквизитам из официального бота);
- сложностей с возвратами.

Если аудитория сайта — преимущественно подростки (Roblox), стоит особенно внимательно отнестись к прозрачности этой схемы или со временем подключить настоящий платёжный шлюз (ЮKassa, CloudPayments, CryptoCloud), который подтверждает оплату автоматически.
